Running the project

The project was created in the vispack tree as required and is compiled same way vispack-build directory is compiled. 

To run project2 go to the directory where the executable is and type ./project2 <imagefile>. After this there are 4 options in the menu, you can choose any operation and  provide some inputs. Once the operation is complete the main menu is shown again, to quit, type 4 or Ctrl+c. To select a new image you need to rerun the project.

Code organization
  • All the functions are in one file project2.cxx. 
  • One main function that takes in the image filename as the parameter.
  • This main function in turn calls one of the menu function corresponding to the operation that user picks.
    • For example: if the user picks 1. Histogram equalization then the corresponding menuHisteq(I) is called.
  • The menu functions takes care asking the user for additional parameters and in turn calling function that performs the required job and then writing the output image or text file (in case of histograms).
    • For the above example, it would prompt the user to enter the number of bins, min and max intensity and the alpha value. And then histeq(I,n,min,max,alpha) is called and writes the output image along with histograms.

Histogram Equalization

Histogram

Histogram of images describe the frequency of the intensity values that occur in an image. The histogram h for a grayscale image is given as follows,

    h[i] = number of pixels in the image with the intensity value i.

I have implemented the above formula in histogram(I, n, min, max). The function takes image, number of bins, minimum intensity and maximum intensity, returns the histogram as a float array. The code moves through all the pixels increasing the corresponding histogram bin value.

Histogram Equalization

The goal of histogram equalization is to find and apply a point operation such that the histogram of the modifies image approximates a uniform distribution. The point operation f(x,y) is obtained from the cumulative histogram H of the original image as follows

f(x,y) = H[g(x,y)] * (n-1) / M*N

where g(x,y) is the old images intensity at (x,y) and f(x,y) is the new image, n is the number of grey levels and M*N is the size of the input image.

The linear blend algorithm is fairly simple. All I do use the following equation

f(x,y) = alpha*new_intensity + (1-alpha)*old_intensity

where alpha is the input blending value, new_intensity is the value calculated using the histogram equalization equation (given above) and the old_intensity is the intensity from the original image.

I have implemented the above algorithm in the function histeq(I, n, min, max, alpha). The function takes image, number of bins, minimum intensity, maximum intensity and an optional blending value as the parameter. It returns the equalized image. The input parameter alpha should be within 0 and 1, if the user enters any other value it is defaulted to be equal to 1.

The code gets the histogram from the implemented histogram() function and calculates the cumulative histogram. It then parses each pixel and recalculates the new intensity value using the above formula. I also clamp the new intensity value to be within min and max. This algorithm requires time O(m*n) for an image of with m*n pixels.

Adaptive Histogram Equalization

Adaptive Histogram Equalization is a much effective algorithm for contrast enhancement than the histogram equalization. I have implemented the AHE(window based) in its basic form. Here, we define a contextual region (I have taken this region to be square), then a histogram of that region is calculated and then the CDF of the histogram is obtained. The central pixel in each region is histogram equalized in that region. Below is the pseudo code for this algorithm.

for every pixel i (with grey level l)in image do
  Initialize array Hist to zero;
  for every contextual pixel j do
     Hist[g(j)] = Hist[g (j)]+1;
  end
      for i = 1 to l
            Hist[i] += Hist[i-1]

     l’ = Hist[l]*L/W^2 
end

where L is the number of grey levels, W is the window size (since its square contextual region the size is W squared), l and l' is the old and new grey level respectively and g(j) is the intensity of the pixel j. The above pseudo code was taken from the paper "A fast implementation of adaptive histogram equalization" by Wang Zhiming and Tao Jianhua.

This algorithm takes a time of O(MxNx(W^2+n)) where MxN is the number of pixels in the image, W is the window size and n is the number of bins. Since I'm only calculating cdf for the original intensity value, the n in the time complexity varies from 1 to n.

I have implemented the above algorithm in the function AHE(I, n, min, max, windowsize). The function takes image, number of bins, minimum intensity, maximum intensity and an optional window size as the parameter. It returns the equalized image. There is an overloaded function AHE(I, n, windowsize) which intern calls the function as AHE(I, n, 0.f, 255.f, windowsize). If window size is negative I take the 1/8th of the image width as the window size. I have done this because in the paper "Adaptive Histogram Equalization and its variation" by Pizer et. al., it is mentioned that mostly a window area between 1/16th and 1/64th of the image area is optimal. Since the images are fairly square, taking the window size as 1/8th of the image width will make the window area to be approximately 1/64th of the image.

To attend to the boundary pixels, the empty contextal region is filled with constant intensity of 0. This gives me a better results around the boundaries compared to no padding on the boundary.

Clipped Local Adaptive Histogram Equalization

Although AHE is effective in enhancing the local signal of the image, it also enhances noise in the image. Contrast enhancement can be defined as the slope of the function mapping from input to output intensity. With histogram equalization the mapping function m(i) is proportional to the cumulative histogram.

m(i) = (Display_range)*(Cumulative_histogram(i)/region_size)

Thus the derivative of m(i) is proportional to histogram(i). So, the trick in CLAHE is to limit the histogram which in turn limits the contrast enhancement.

Adding this portion into AHE is pretty straight forward. I take the clipping_limit value from the user (0 to 1, direct value for limiting the pdf of histogram) and remove the pixels that are above the clipping limit and redistribute the pixels uniformly across all bins, including those that have already reached the limit.

I have implemented the above algorithm in the function CLAHE(I, n, min, max, clip_limit, windowsize). The function takes image, number of bins, minimum intensity, maximum intensity, clipping limit and an optional window size as the parameter. It returns the equalized image. There is an overloaded function CLAHE(I, n, clip_limit, windowsize) which intern calls the function as CLAHE(I, n, 0.f, 255.f, clip_limit, windowsize). The clip_limit parameter passed here should be the CL_input*window_size^2, since the histograms are not normalized.

The project was executed in 2GHz desktop with 3 GB ram. The time for execution is reported along with the images.

On all images CLAHE gave better contrast enhancement than AHE or global histogram equalization. Clip values between 0.01 and 0.05 seems most effective as lower clip values will give an image similar to the original and higher values will gives images equivalent to those of AHE's.
